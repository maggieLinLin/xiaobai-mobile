# 桌面端尺寸修复

## 问题描述

用户反馈：
1. 手机模拟器在电脑上显示太大，甚至看不到边框
2. 边框从蓝色变成了黑色
3. 手机正常尺寸绝对比这个小

## 根本原因

之前的设置：
```css
/* ❌ 问题 1: 尺寸太高 */
width: 375px;
height: 812px;  /* iPhone X 的高度，过高 */

/* ❌ 问题 2: 桌面端 1:1 显示 */
@media (min-width: 769px) {
    transform: scale(1);  /* 完全不缩放，导致太大 */
}

/* ❌ 问题 3: 黑色边框 */
border: 10px solid #1c1c1e;  /* 深灰色/黑色 */
```

**问题分析：**
- iPhone X (812px 高) 在桌面端 1:1 显示时，占据整个屏幕高度
- 普通笔记本（1080p）高度只有 1080px，手机占 812px，几乎看不到边框
- 没有缩小，导致不符合真实手机在桌面上的视觉比例

## 解决方案

### 核心改进

1. **改用 iPhone 8 尺寸 (375x667)**：更短，更常见
2. **桌面端缩小到 70%**：更接近真实手机在桌面的视觉比例
3. **恢复蓝色边框**：`#78B9DC`
4. **优化圆角和边框宽度**

### 修改详情

```css
#phone-frame {
    /* ✅ 改为 iPhone 8 尺寸 */
    width: 375px;
    height: 667px;  /* 比 812px 短 145px */
    
    /* ✅ 恢复蓝色边框 */
    border: 10px solid #78B9DC;
    border-radius: 32px;
}

/* ✅ 桌面端：缩小到 70% */
@media (min-width: 769px) {
    #phone-frame {
        transform: scale(0.7);
    }
}

/* ✅ 平板端：最大 0.9 */
@media (max-width: 768px) and (min-width: 481px) {
    #phone-frame {
        transform: scale(min(
            calc((100vw - 40px) / 375), 
            calc((100vh - 40px) / 667), 
            0.9  /* 限制最大 0.9 */
        ));
    }
}

/* ✅ 手机端：自适应，边框更细 */
@media (max-width: 480px) {
    #phone-frame {
        width: 375px;
        height: 667px;
        border-radius: 32px;
        border-width: 8px;  /* 8px 代替 10px */
        transform: scale(min(
            calc((100vw - 20px) / 375), 
            calc((100vh - 20px) / 667)
        ));
    }
}

/* ✅ 小屏手机：边框 6px */
@media (max-width: 380px) {
    #phone-frame {
        border-radius: 28px;
        border-width: 6px;  /* 更细 */
    }
}
```

## 为什么选择 70% 缩放？

### 视觉比例计算

真实手机在桌面上的视觉感受：
- 真实 iPhone 8: 138.4mm 高
- 24 寸显示器: 约 300mm 高
- 比例: 138.4 / 300 ≈ 0.46 (46%)

但考虑到：
1. 用户需要看清内容
2. 过小会影响操作
3. 保留足够的视觉空间

**选择 70% 是最佳平衡点：**
- 不会太大（占满屏幕）
- 不会太小（看不清）
- 符合真实手机的视觉感受
- 四周留白充足
